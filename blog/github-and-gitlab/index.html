GitHub and GitLab

2022-01-24

Since I’ve started re-entering the world of programming, I’ve been exploring options for how to organize and manage my projects.

Github and Gitlab are the biggest players in this field. I have much more experience with GitLab and have been using it since ~2013, back when it was the option for free private repos. My profile(s) included stuff like my dotfiles, school research projects, notes, and shitty scripts; all in private repos of course. My former employer also used GitLab, so I was able to get comfortable with using it in an enterprise environment. I like GitLab, and up til now, I had no reason to consider switching. The thing is, GitHub is big… like really big, and exposure is something I need to worry about if I am thinking about working in the industry.

Quick Comparison

GitHub

  • big
  • closed-source; microsoft-owned
  • simple and convinient
  • app marketplace to extend functionality
  • lot of features locked behind paywall
  • GitHub flow
  • team plan $4/month

GitLab

  • small
  • open-source
  • free self-hosted
  • feature complete platform with some integrations
  • better issue tracking and project management
  • GitLab flow
  • premium plan $19/month

Thoughts

GitLab seems objectively better as a platform. It tickles my open-source bone and its issue tracking and project management tools are what I’m comfortable with. But if I wan’t to start contributing to open source projects or have my projects noticed, the larger pool of developers on GitHub makes the decision a no-brainer. I already prefer GitHub flow for personal projects as well. The fact that Microsoft owns GitHub is concerning, but I use VSCode (or VSCodium), so I can’t really say much; neither platform is free from controversy. The benefits of GitHub outweigh my preferences. In any case, it’s worth a try.

I’m going to start to migrate to GitHub, but I don’t want to abandon GitLab, so I’ll keep a mirror of my repos on there. GitLab has a built-in option for mirroring, so I don’t have to worry about managing multiple remotes.

I’m moving my site back onto GitHub Pages too, just so all of my stuff is consolidated.
